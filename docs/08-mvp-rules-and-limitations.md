
# MVP Rules and Limitations

## هدف
جلوگیری از بزرگ‌شدن بیش از حد پروژه و حفظ تمرکز روی نسخه قابل فروش اولیه.

## قوانین MVP

### 1. پاسخ خودکار فقط متنی است
- پاسخ تصویری، صوتی و دکمه‌ای در MVP وجود ندارد

### 2. فقط دو نوع Match
- `exact`
- `contains`

### 3. فقط یک اقدام فیلتر
- `delete_message`

### 4. فقط گزارش پایه
- آمار روزانه و هفتگی
- بدون داشبورد تحلیلی پیشرفته

### 5. فقط اولین قانون پاسخ اجرا می‌شود
- برای جلوگیری از چند پاسخ هم‌زمان

### 6. تنظیمات per-channel
- هر کانال یا گروه تنظیمات مستقل خود را دارد

### 7. Scheduler ساده
- Polling در بازه زمانی ثابت
- بدون زمان‌بندی پیچیده

---

## مواردی که بعداً اضافه می‌شوند
- Regex Matching
- Multi-step Auto Reply
- Warning / Mute / Ban
- گزارش‌های تحلیلی
- نقش‌های پیشرفته
- Billing و Payment
- Webhook retries پیشرفته
- Rule Builder حرفه‌ای

---

## معیار تصمیم‌گیری
اگر قابلیتی:
- توسعه را کند می‌کند
- فروش اولیه را عقب می‌اندازد
- برای اولین مشتری ضروری نیست

باید از MVP حذف شود.
