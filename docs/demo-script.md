# A05 Demo Script

## Goal

Show that `A05` is deployable, operational, and useful without needing a frontend panel.

## 5-Minute Flow

1. Introduce the service as a self-hosted Rubika automation backend
2. Show `GET /api/v1/health` returning success
3. Show auth login or token flow
4. Show one workspace and one channel path
5. Show a scheduled post example
6. Show an auto-reply or filter example
7. Close with deployment and support model

## Demo Evidence

- Health endpoint responds successfully
- API docs are reachable in non-production debug environments
- One real or seeded workflow can be explained end-to-end
- Deployment docs and handover checklist are ready

## Close

End the demo by clarifying that the buyer is purchasing a deployable backend service with repeatable delivery, not an unfinished generic SaaS promise.
