# Background Jobs

## هدف
انجام عملیات زمان‌بر و تکرارشونده خارج از مسیر اصلی درخواست کاربر.

## Jobهای MVP

### 1. Scheduled Post Dispatcher
وظیفه:
- بررسی پست‌های موعدرسیده
- ارسال آن‌ها
- ثبت نتیجه

#### ورودی
- رکوردهای `scheduled_posts` با شرایط:
  - `status = scheduled`
  - `scheduled_at <= now()`

#### خروجی
- به‌روزرسانی وضعیت پست
- ایجاد `post_logs`
- ایجاد `activity_logs`

#### گام‌ها
1. انتخاب پست‌های موعدرسیده
2. قفل‌کردن منطقی رکورد
3. تغییر وضعیت به `sending`
4. ارسال از طریق Rubika API
5. ثبت موفقیت یا شکست
6. به‌روزرسانی وضعیت نهایی

#### فرکانس اجرا
- هر 30 ثانیه یا هر 1 دقیقه

---

### 2. Retry Failed Posts
در MVP می‌تواند غیرفعال باشد یا خیلی ساده اجرا شود.

#### سیاست ساده
- اگر `retry_count < 3`
- و خطا موقتی باشد
- مجدداً تلاش شود

---

### 3. Daily Report Aggregator
برای MVP این Job اختیاری است.
می‌توان گزارش را مستقیم از دیتابیس تولید کرد.

اگر تعداد داده بالا رفت:
- شبانه آمار خلاصه تولید شود
- در جدول summary ذخیره شود

---

## نکات اجرایی
- Worker باید جدا از API Process اجرا شود
- عملیات ارسال باید idempotent باشد تا دوباره‌ارسالی کمتر شود
- در صورت شکست API روبیکا، لاگ کامل خطا ثبت شود
- برای رکوردهای در حال ارسال، از status=`sending` استفاده شود

---

## قفل‌گذاری منطقی
برای جلوگیری از ارسال دوباره یک پست:
- قبل از ارسال، status از `scheduled` به `sending` تغییر کند
- اگر Worker دیگری همان رکورد را دید، دیگر آن را برندارد
