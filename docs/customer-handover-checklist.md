# A05 Customer Handover Checklist

## Access

- Customer has server access path
- Customer has domain and DNS ownership clarity
- Customer receives `.env` ownership instructions without secrets being committed

## Runtime

- API service path documented
- Worker and scheduler service names documented
- Healthcheck URL shared
- Backup and restore runbook shared

## Operations

- Restart commands shared
- Log locations shared
- Webhook registration state confirmed
- Secret rotation owner identified

## Support

- Support boundary documented
- Escalation contact documented
- Maintenance retainer status documented

## Acceptance

- Customer confirms healthcheck works
- Customer confirms at least one operator can follow the runbook
- Customer confirms handover artifacts were received
