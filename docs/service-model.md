# A05 Service Model

## Positioning

`A05` is a self-hostable Rubika bot automation backend delivered as a managed engineering service, not as a public low-ticket SaaS.

## Primary Offer

- Managed deployment for one customer workspace stack
- Setup of API, worker, scheduler, PostgreSQL, Redis, and webhook
- Configuration handoff and operator runbook
- Limited launch support after go-live

## Allowed Delivery Modes

1. Managed deployment on customer VPS
2. Self-hosted delivery with guided handoff
3. Custom enterprise implementation with scoped add-ons

## Default Ownership Model

- Customer owns server, domain, and Rubika bot identity
- Delivery team owns implementation, deployment, and initial stabilization
- Ongoing support is optional and should be contracted separately

## Core Deliverables

- Running backend stack
- Validated `.env`
- Registered webhook
- Working healthcheck
- Verified worker and scheduler processes
- Deployment and handover documents

## Support Boundary

- Includes initial install, configuration, smoke verification, and documentation
- Does not include custom frontend, CRM, billing, or unrelated product development

## Success Definition

The service is complete only when the customer can operate a deployed Rubika bot backend with documented setup, known limits, and a clear support path.
